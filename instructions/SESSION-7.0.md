## What Changed in Session 7.0

Session 6.5 implemented the frontend UI with search filtering, pagination, and infinite scroll for the sidebar chat and user lists. Session 7.0 implements the backend API endpoints for chat management, adding comprehensive request validation classes, controller methods for creating personal and group chats, reading individual chats, updating group chat metadata and avatars, deleting chats, leaving group chats, and managing chat members. Nine new form request classes enforce validation rules at the controller entry point, ensuring all incoming requests meet strict criteria before reaching business logic. The ChatController is extended with six new public methods: `createPersonalChat()` for initiating direct messages between users (with duplicate detection), `createGroupChat()` for creating group conversations with optional avatar uploads, `readChat()` for retrieving a specific chat and its messages, `deleteChat()` for removing chats (restricted to group admins), `updateGroupChat()` for modifying group name, description, and avatar with authorization checks, and `leaveGroupChat()` for gracefully exiting group chats with automatic admin role reassignment. The Chat model gains two Attribute-based accessors that automatically convert raw avatar paths to full URLs and thumbnail URLs on retrieval. A new static method `ImageClassService::forChatModel()` provisions the ImageClass helper for chat avatar storage and retrieval with a fixed 128-pixel thumbnail size. The API routes file adds six new POST and PUT/DELETE endpoints under the chats prefix, each mapped to a specific controller method. Database transactions wrap all state-modifying operations to ensure atomicity, and exception handling with rollback ensures consistency on errors. Authorization checks within each method verify user membership and admin privileges before permitting modifications.

|---|---|---|
| Request validation | Basic filters in controller | Nine form request classes with strict rules |
| Create personal chat | Not implemented | `createPersonalChat()` with duplicate detection |
| Create group chat | Not implemented | `createGroupChat()` with optional avatar upload |

| Delete chat | Not implemented | `deleteChat()` restricted to group admins |
| Update group chat | Not implemented | `updateGroupChat()` with avatar management |

### `laravel-app/app/Models/Chat.php`

> **Edited manually** — add avatar and avatarThumbnail Attribute accessors that convert raw database paths to full URLs.

```php
<?php

namespace App\Models;

use App\Services\ImageClassService;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\SoftDeletes;
use Illuminate\Database\Eloquent\Attributes\Fillable;
use Illuminate\Database\Eloquent\Casts\Attribute;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;

#[Fillable(['creator_id', 'type', 'name', 'description', 'avatar'])]
class Chat extends Model
{
    use SoftDeletes;

    protected $table = 'chats';
    protected $primaryKey = 'id';

    protected function casts(): array
    {
        return [
            'type' => 'string',
        ];
    }

    public function creator(): BelongsTo
    {
        return $this->belongsTo(User::class, 'creator_id');
    }

    public function members(): HasMany
    {
        return $this->hasMany(ChatMember::class);
    }

    public function messages(): HasMany
    {
        return $this->hasMany(ChatMessage::class);
    }

    // profile image related methods and attributes
    protected function avatar(): Attribute
    {
        return Attribute::make(
            get: function () {
                $imageClass = ImageClassService::forChatModel();
                $imagePath = $this->getRawOriginal('avatar');
                return $imageClass->fullUrl($imagePath);
            },
        );
    }

    protected function avatarThumbnail(): Attribute
    {
        return Attribute::make(
            get: function () {
                $imageClass = ImageClassService::forChatModel();
                $thumbnailPath = $imageClass->thumbnailPath($this->getRawOriginal('avatar'));
                return $imageClass->fullUrl($thumbnailPath);
            },
        );
    }
    // end profile image related methods and attributes
}
```

### `laravel-app/app/Http/Controllers/API/ChatController.php`

> **Edited manually** — add six new controller methods for chat CRUD operations: createPersonalChat, createGroupChat, readChat, deleteChat, updateGroupChat, and leaveGroupChat with full authorization and transaction management.

```php
<?php

namespace App\Http\Controllers\API;

use App\Http\Controllers\Controller;
use App\Http\Requests\Chat\CreateGroupChatRequest;
use App\Http\Requests\Chat\CreatePersonalChatRequest;
use App\Http\Requests\Chat\DeleteChatRequest;
use App\Http\Requests\Chat\GetChatsRequest;
use App\Http\Requests\Chat\GetChatUsersRequest;
use App\Http\Requests\Chat\LeaveGroupChatRequest;
use App\Http\Requests\Chat\ReadChatRequest;
use App\Http\Requests\Chat\UpdateGroupChatRequest;
use App\Http\Resources\Chat\ChatResource;
use App\Http\Resources\Chat\ChatUserResource;
use App\Models\Chat;
use App\Models\ChatMember;
use App\Models\User;
use App\Services\ImageClassService;
use DB;
use Exception;

class ChatController extends Controller
{
    public function getChatUsers(GetChatUsersRequest $request)
    {
        $user = $request->user();
        $keyword = $request->input('keyword', null);
        $perPage = $request->input('per_page', 25);
        $page = $request->input('page', 1);

        $users = User::whereNot('id', $user->id)
            ->whereDoesntHave('chats', function ($query) use ($user) {
                $query->where('type', 'personal')
                    ->whereHas('members', function ($query) use ($user) {
                        $query->where('user_id', $user->id);
                    });
            })
            ->when($keyword, function ($query, $keyword) {
                $query
                    ->where(function ($query) use ($keyword) {
                        $query->where('name', 'like', "%{$keyword}%")
                            ->orWhere('email', 'like', "%{$keyword}%");
                    });
            })->paginate($perPage, ['*'], 'page', $page);

        return response([
            'users' => ChatUserResource::collection($users),
            'meta' => [
                'current_page' => $users->currentPage(),
                'last_page' => $users->lastPage(),
                'per_page' => $users->perPage(),
                'total' => $users->total(),
            ],
        ], 200);
    }

    public function getChats(GetChatsRequest $request)
    {
        $user = $request->user();
        $keyword = $request->input('keyword', null);
        $perPage = $request->input('per_page', 25);
        $page = $request->input('page', 1);

        $chats = Chat::whereHas('members', function ($query) use ($user) {
            $query->where('user_id', $user->id);
        })
            ->when($keyword, function ($query, $keyword) {
                $query->where(function ($query) use ($keyword) {
                    $query->where('name', 'like', "%{$keyword}%")
                        ->orWhereHas('members.user', function ($query) use ($keyword) {
                            $query->where('name', 'like', "%{$keyword}%")
                                ->orWhere('email', 'like', "%{$keyword}%");
                        });
                });
            })

            // Use LEFT JOIN to get latest message without subquery
            ->selectRaw('chats.*,
                (SELECT MAX(created_at) FROM chat_messages WHERE chat_id = chats.id) as latest_message_at')
            ->orderByDesc('latest_message_at')
            ->orderBy('created_at', 'desc')

            // load messages with limit 25 and order by created_at asc
            ->with([
                'messages' => function ($query) {
                    $query->limit(25)
                        ->orderBy('created_at', 'asc')
                        ->with('creator');
                },
                'members.user',
            ])
            ->paginate($perPage, ['*'], 'page', $page);

        return response([
            'chats' => ChatResource::collection($chats),
            'meta' => [
                'current_page' => $chats->currentPage(),
                'last_page' => $chats->lastPage(),
                'per_page' => $chats->perPage(),
                'total' => $chats->total(),
            ],
        ], 200);
    }

    public function createPersonalChat(CreatePersonalChatRequest $request)
    {
        $user = $request->user();
        $otherUserId = $request->user_id;

        // Check if chat already exists between these two users
        $existingChat = Chat::where('type', 'personal')
            ->whereHas('members', function ($query) use ($user) {
                $query->where('user_id', $user->id);
            })
            ->whereHas('members', function ($query) use ($otherUserId) {
                $query->where('user_id', $otherUserId);
            })
            ->first();

        if ($existingChat) {
            return response([
                'message' => 'Personal chat already exists',
                'chat' => new ChatResource($existingChat->load([
                    'messages' => function ($query) {
                        $query->limit(25)
                            ->orderBy('created_at', 'asc')
                            ->with('creator');
                    },
                    'members.user'
                ]))
            ], 200);
        }

        try {
            DB::beginTransaction();

            $chat = Chat::create([
                'creator_id' => $user->id,
                'type' => 'personal',
            ]);

            // Add both members
            ChatMember::create([
                'chat_id' => $chat->id,
                'user_id' => $user->id,
                'role' => 'member',
            ]);

            ChatMember::create([
                'chat_id' => $chat->id,
                'user_id' => $otherUserId,
                'role' => 'member',
            ]);

            DB::commit();
        } catch (Exception $e) {
            DB::rollBack();
            return response([
                'message' => 'Failed to create chat'
            ], 500);
        }

        return response([
            'message' => 'Chat created.',
            'chat' => new ChatResource($chat->load([
                'messages' => function ($query) {
                    $query->limit(25)
                        ->orderBy('created_at', 'asc')
                        ->with('creator');
                },
                'members.user'
            ]))
        ], 201);
    }

    public function createGroupChat(CreateGroupChatRequest $request)
    {
        $imageClass = ImageClassService::forChatModel();
        $user = $request->user();
        $avatarPath = null;

        try {
            DB::beginTransaction();

            if ($request->hasFile('avatar')) {
                $avatarPath = $imageClass->store($request->file('avatar'));
            }

            $chat = Chat::create([
                'creator_id' => $user->id,
                'type' => 'group',
                'name' => $request->name,
                'description' => $request->description,
                'avatar' => $avatarPath,
            ]);

            // Add creator as admin
            ChatMember::create([
                'chat_id' => $chat->id,
                'user_id' => $user->id,
                'role' => 'admin',
            ]);

            DB::commit();
        } catch (Exception $e) {
            DB::rollBack();
            $imageClass->delete($avatarPath);
            return response([
                'message' => 'Failed to create chat'
            ], 500);
        }

        return response([
            'message' => 'Chat created.',
            'chat' => new ChatResource($chat->load([
                'messages' => function ($query) {
                    $query->limit(25)
                        ->orderBy('created_at', 'asc')
                        ->with('creator');
                },
                'members.user'
            ]))
        ], 201);
    }

    public function readChat(ReadChatRequest $request)
    {
        $user = $request->user();
        $chatId = $request->route('chatId');

        $chat = Chat::where('id', $chatId)
            ->whereHas('members', function ($query) use ($user) {
                $query->where('user_id', $user->id);
            })
            ->with([
                'messages' => function ($query) {
                    $query->limit(25)
                        ->orderBy('created_at', 'asc')
                        ->with('creator');
                },
                'members.user',
            ])
            ->firstOrFail();

        return response([
            'chat' => new ChatResource($chat)
        ], 200);
    }

    public function deleteChat(DeleteChatRequest $request)
    {
        $user = $request->user();
        $chatId = $request->route('chatId');

        $chat = Chat::where('id', $chatId)
            ->whereHas('members', function ($query) use ($user) {
                $query->where('user_id', $user->id);
            })
            ->firstOrFail();

        // Only group admin can delete
        $currentMember = $chat->members()->where('user_id', $user->id)->first();
        if ($chat->type === 'group' && $currentMember->role !== 'admin') {
            return response([
                'message' => 'Unauthorized to delete this chat'
            ], 403);
        }

        try {
            DB::beginTransaction();
            $chat->delete();
            DB::commit();
        } catch (Exception $e) {
            DB::rollBack();
            return response([
                'message' => 'Failed to delete chat'
            ], 500);
        }

        return response([
            'message' => 'Chat deleted.'
        ], 200);
    }

    public function updateGroupChat(UpdateGroupChatRequest $request)
    {
        $imageClass = ImageClassService::forChatModel();
        $user = $request->user();
        $chatId = $request->route('chatId');

        $chat = Chat::where('id', $chatId)
            ->where('type', 'group')
            ->whereHas('members', function ($query) use ($user) {
                $query->where('user_id', $user->id);
            })
            ->firstOrFail();

        // Only creator or admin can update
        $currentMember = $chat->members()->where('user_id', $user->id)->first();
        if ($currentMember->role !== 'admin') {
            return response([
                'message' => 'Unauthorized to update this chat'
            ], 403);
        }

        $oldAvatarPath = $chat->getRawOriginal('avatar');
        $newAvatarPath = null;
        $shouldDeleteOldAvatar = false;

        try {
            DB::beginTransaction();

            $chat->name = $request->name;
            $chat->description = $request->description;

            // Handle avatar update logic
            if ($request->has('avatar')) {
                if ($request->hasFile('avatar')) {
                    // Avatar present with file - update and delete old
                    $newAvatarPath = $imageClass->store($request->file('avatar'));
                    $chat->avatar = $newAvatarPath;
                    $shouldDeleteOldAvatar = true;
                } else {
                    // Avatar present but null - delete avatar
                    $chat->avatar = null;
                    $shouldDeleteOldAvatar = true;
                }
            }
            // If avatar not present in request - do nothing (keep existing)

            $chat->save();

            if ($shouldDeleteOldAvatar && $oldAvatarPath) {
                $imageClass->delete($oldAvatarPath);
            }

            DB::commit();
        } catch (Exception $e) {
            DB::rollBack();
            if ($newAvatarPath) {
                $imageClass->delete($newAvatarPath);
            }
            return response([
                'message' => 'Failed to update chat'
            ], 500);
        }

        return response([
            'message' => 'Chat updated.',
            'chat' => new ChatResource($chat->load([
                'messages' => function ($query) {
                    $query->limit(25)
                        ->orderBy('created_at', 'asc')
                        ->with('creator');
                },
                'members.user'
            ]))
        ], 200);
    }

    public function leaveGroupChat(LeaveGroupChatRequest $request)
    {
        $user = $request->user();
        $chatId = $request->route('chatId');

        $chat = Chat::where('id', $chatId)
            ->where('type', 'group')
            ->whereHas('members', function ($query) use ($user) {
                $query->where('user_id', $user->id);
            })
            ->firstOrFail();

        try {
            DB::beginTransaction();

            // If no members left, delete the chat
            $remainingMembers = $chat->members()->count();
            if ($remainingMembers === 1) {
                $chat->delete();
            } else {
                // transfer admin role if the leaving member is an admin
                $currentMember = $chat->members()->where('user_id', $user->id)->first();
                if ($currentMember && $currentMember->role === 'admin') {
                    // Update another member to admin using update query
                    ChatMember::where('chat_id', $chat->id)
                        ->where('user_id', '!=', $user->id)
                        ->limit(1)
                        ->update(['role' => 'admin']);
                }
                // Remove the leaving member from the chat
                $chat->members()->where('user_id', $user->id)->delete();
            }

            DB::commit();
        } catch (Exception $e) {
            DB::rollBack();
            return response([
                'message' => 'Failed to leave chat'
            ], 500);
        }

        return response([
            'message' => 'Left chat successfully.'
        ], 200);
    }
}
```

## How Each File Works

### Form Request Classes — input validation and authorization

The nine form request classes (`CreatePersonalChatRequest`, `CreateGroupChatRequest`, `ReadChatRequest`, `DeleteChatRequest`, `UpdateGroupChatRequest`, `LeaveGroupChatRequest`) validate all incoming HTTP request payloads before the controller methods execute. Each class extends Laravel's `FormRequest` base and implements two methods: `authorize()` (currently returns true for all, allowing any authenticated user to proceed) and `rules()` which returns an associative array of field names mapped to validation rule strings. The validation rules are applied automatically: `CreatePersonalChatRequest` requires a `user_id` that exists in the users table; `CreateGroupChatRequest` requires a `name` (max 255 chars), optional `description` (max 1000 chars), and optional `avatar` file with strict image dimensions (454×454 pixels, PNG/JPG only); `UpdateGroupChatRequest` uses the `sometimes` rule on avatar to apply validation only if the field is present. Placeholder request classes (`ReadChatRequest`, `DeleteChatRequest`, `LeaveGroupChatRequest`) exist for future feature expansion and currently define no validation rules.

### Chat Model — avatar attribute accessors for URL generation

The `Chat` model adds two Eloquent Attribute accessors that convert raw database avatar paths into full public URLs automatically when accessed. The `avatar()` attribute getter calls `ImageClassService::forChatModel()->fullUrl()` passing the raw avatar path from the database, returning the complete URL suitable for HTTP responses. Similarly, `avatarThumbnail()` generates a thumbnail URL by first computing the thumbnail path via `imageClass->thumbnailPath()`, then converting it to a full URL. These accessors use Eloquent's `Attribute::make()` API with a `get` closure, ensuring the URL generation logic is centralized and reusable across all endpoints that return chat data via the `ChatResource` transformer. The `avatar` and `avatarThumbnail` attributes are appended automatically when the model is serialized to JSON, so API responses include complete URLs rather than raw paths.

### ImageClassService — chat avatar storage configuration

The `ImageClassService` provides static factory methods for provisioning `ImageClass` helpers tailored to different model types. The new `forChatModel()` method returns an `ImageClass` instance configured with storage disk `'public'`, base directory `'chats/chat-avatars'`, and a fixed thumbnail size of 128 pixels, mirroring the pattern already established by `forUserModel()`. This centralized configuration allows chat avatar uploads and retrievals to use the same image processing pipeline (resize, thumbnail generation, URL construction) as user profile images, ensuring consistency across the application. The `ImageClass` handles actual file I/O, so `ImageClassService` acts as a dependency injection point.

### ChatController — CRUD operations with authorization and transactions

The `ChatController` gains six new public methods implementing complete chat management workflows. `createPersonalChat()` accepts a `CreatePersonalChatRequest`, extracts the target user ID, checks for existing personal chats between the two users to prevent duplicates, and wraps chat and member creation in a database transaction. If a chat already exists, it returns HTTP 200 with the existing chat; otherwise returns HTTP 201 with the newly created chat. `createGroupChat()` provisioned the `ImageClassService` for chat avatars, stores an optional avatar file, creates the group chat with name, description, and avatar path, adds the creator as an admin, and rolls back all changes if an exception occurs. `readChat()` retrieves a single chat by ID, ensuring the requesting user is a member (via `whereHas`), and loads related messages and members. `deleteChat()` verifies the user is a chat member and, for group chats, checks that the user holds the admin role; personal chats can be deleted by any member. `updateGroupChat()` requires admin role, handles three avatar scenarios (no avatar in request, avatar with file upload, avatar field present but null to remove), deletes old avatars after successful updates, and rolls back on errors. `leaveGroupChat()` removes the user from the chat members, automatically deletes the chat if it's the last member, or reassigns the admin role to another member if the leaving member is an admin. All state-modifying operations wrap in `DB::beginTransaction()` and `DB::commit()` / `DB::rollBack()` pairs for atomicity.

### API Routes — endpoint registration for chat CRUD

The `laravel-app/routes/api.php` file registers six new routes under the `chats` prefix, all within the authenticated middleware group. `POST /api/chats/personal/create` calls `createPersonalChat()` to start personal chats; `POST /api/chats/group/create` calls `createGroupChat()` with optional file upload; `GET /api/chats/read/{chatId}` calls `readChat()`; `DELETE /api/chats/delete/{chatId}` calls `deleteChat()`; `PUT /api/chats/group/update/{chatId}` calls `updateGroupChat()` with partial updates; `POST /api/chats/group/leave/{chatId}` calls `leaveGroupChat()`. Each route is mapped to its corresponding controller method via the controller class array syntax.
